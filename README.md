2. 사운드 효과 및 클릭 이벤트 구현
주요 기능

버튼을 클릭하면 “축하합니다” 메시지 출력
버튼 클릭 시 효과음 재생
버튼이 도망갈 때도 별도의 효과음 재생
이를 통해 사용자에게 시각적뿐만 아니라 청각적인 피드백도 제공하도록 구현하였다.

어려웠던 점
처음에는 mp3 파일을 사용했지만 SoundPlayer에서 재생되지 않는 문제가 발생하였다. 이를 해결하기 위해 wav 파일로 변환해야 했으며, 파일 경로를 지정하는 과정에서도 오류가 발생하였다. 특히 경로에 공백이나 한글이 포함될 경우 실행이 되지 않는 문제가 있어 정확한 경로 설정의 중요성을 알게 되었다.

3. 점수 및 게임 요소 추가
주요 기능

버튼 클릭 시 점수 100점 증가
버튼을 놓칠 경우 점수 10점 감소
버튼 클릭 시 점점 크기가 작아지도록 설정
폼의 제목 표시줄에 현재 점수 출력
단순한 버튼 동작에서 벗어나 점수 시스템을 도입하여 게임 형태로 확장하였다.

어려웠던 점
점수 증가와 감소를 동시에 처리하는 과정에서 조건을 잘못 설정하면 점수가 비정상적으로 변하는 문제가 발생하였다. 또한 버튼 크기를 줄이는 과정에서 너무 작아져 클릭이 어려워지는 문제가 있었기 때문에 최소 크기를 제한하는 조건을 추가해야 했다.

4. 게임 오버 및 다시 시작 기능 구현
주요 기능

버튼을 20번 놓칠 경우 Game Over 메시지 출력
게임 종료 시 버튼 비활성화
“다시 시작” 버튼 클릭 시 게임 초기화
점수 초기화
버튼 크기 복구
버튼 위치를 중앙으로 이동
게임의 시작과 종료 흐름을 구성하여 하나의 완성된 프로그램 형태로 만들었다.

어려웠던 점
다시 시작 기능을 구현하는 과정에서 버튼 크기가 원래대로 복구되지 않는 문제가 발생하였다. 이를 해결하기 위해 초기 버튼 크기를 변수에 저장한 뒤, 리셋 시 해당 값을 다시 적용하도록 수정하였다. 또한 이벤트 연결이 잘못되어 의도하지 않은 동작이 발생하는 문제도 있었는데, 이를 통해 이벤트 처리의 중요성을 이해하게 되었다.

느낀 점
이번 과제를 통해 이벤트 기반 프로그래밍의 개념을 실제로 적용해볼 수 있었다. 특히 MouseEnter와 Click 이벤트를 활용하여 사용자와 상호작용하는 프로그램을 구현하면서, 간단한 기능도 다양한 요소를 추가하면 하나의 게임처럼 확장될 수 있다는 점이 인상 깊었다. 또한 디버깅 과정에서 오류의 원인을 분석하고 해결하는 경험을 통해 문제 해결 능력을 향상시킬 수 있었다.
